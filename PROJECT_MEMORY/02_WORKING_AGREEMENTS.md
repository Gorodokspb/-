# Working Agreements

Project collaboration rules:
- favor fixes that preserve the live business workflow;
- optimize for reliability before elegance;
- prefer targeted edits over sweeping rewrites;
- verify behavior against real user flow whenever possible.

Memory rules:
- every meaningful coding session should leave a note in `06_SESSION_LOG.md`;
- every change of priority should be reflected in `07_NEXT_STEPS.md`;
- every decision that should survive chat history must be copied into `05_DECISIONS.md`.

Scope rules:
- treat manual workflow pain as higher priority than cosmetic cleanup;
- avoid speculative architecture work until current operational bugs are under control;
- protect the estimate-first flow.

Definition of "important enough to remember":
- product rules;
- file ownership and boundaries;
- path and storage assumptions;
- regression-prone UI behavior;
- unfinished work with clear next action;
- blockers discovered during testing.
