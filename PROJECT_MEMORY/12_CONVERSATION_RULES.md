# Conversation Rules

Goal:
- convert important chat outcomes into durable project memory so office and home sessions can resume without relying on chat history.

What should be written down after a meaningful task:
- whether the work happened from `office` or `home`;
- what task we worked on;
- what the user asked for or changed in priorities;
- what files changed;
- what user-facing behavior changed;
- what was verified manually or not verified;
- what the next concrete action is;
- any new decision or constraint discovered during the chat.

Office/home continuity rule:
- if a discussion changes project direction, update memory in the same session;
- do not leave important context only in chat when the next session may happen on another machine;
- store the durable outcome of the dialogue in enough detail that the user does not need to retell it later.

Compression rule:
- do not paste whole conversations;
- store the durable outcome, not the transcript;
- summarize the request, decision, implementation, verification, and next action clearly enough to restore context fast.

Good session note format:
- machine context
- task summary
- key edits
- verification status
- remaining risk
- next action

Reminder:
- chat history is helpful, but the project memory files are the system of record.
